📚 Cloudflare DNS Setup Guide

Complete beginner-friendly guide to set up custom domains for GitHub Pages using Cloudflare

What is Cloudflare and DNS?

Cloudflare is a service that manages your domain's DNS (Domain Name System). Think of DNS as a phone book for the internet - it translates human-readable domain names (like claude-practice.devuard.com) into computer-readable addresses (like GitHub's servers).

Why use Cloudflare?

  • ✅ Easy-to-use dashboard for managing all your domains
  • ✅ Free plan with excellent features
  • ✅ Reliable and fast DNS resolution worldwide
  • ✅ Additional security features
  • ✅ Perfect for connecting multiple GitHub Pages sites to one domain

Prerequisites

Before you start, make sure you have:

  • ✅ A GitHub account with a repository
  • ✅ GitHub Pages enabled on your repository
  • ✅ A domain name (e.g., devuard.com)
  • ✅ A Cloudflare account (free)
  • ✅ Your domain added to Cloudflare

Step 1: Verify Your Repository is Ready

Before configuring DNS, make sure your GitHub Pages site is live:

How to check if GitHub Pages is enabled:

  1. Go to your GitHub repository
  2. Click on Settings (gear icon)
  3. Scroll down to Pages section on the left sidebar
  4. You should see a blue banner saying "Your site is published at https://YOUR_USERNAME.github.io/REPO_NAME"
  5. If not, select "Deploy from a branch" and choose your branch (master/main)

✅ Success: You can now access your site at the GitHub Pages URL

Step 2: Add Your Domain to Cloudflare

If you haven't already added your domain to Cloudflare:

  1. Go to Cloudflare Dashboard
  2. Click Add a Site
  3. Enter your domain name (e.g., devuard.com)
  4. Choose the Free plan
  5. Follow the steps to update your domain's nameservers at your registrar
  6. Wait for DNS propagation (can take up to 48 hours, usually faster)

📝 Note: Nameservers are like the address of the DNS service. You're telling your registrar to use Cloudflare's nameservers.

Step 3: Create a CNAME Record in Cloudflare

This is the key step that connects your subdomain to GitHub Pages!

What is a CNAME Record?

A CNAME (Canonical Name) record is like a forwarding address. It says "when someone visits subdomain.devuard.com, send them to USERNAME.github.io"

How to create a CNAME record:

  1. Go to Cloudflare Dashboard
  2. Select your domain (devuard.com)
  3. Click on DNS in the left menu
  4. Click + Add record button
  5. Fill in the form:
    Type: CNAME Name: claude-practice (or whatever subdomain you want) Target: eduard15x.github.io (your GitHub Pages URL) TTL: Auto Proxy status: Proxied (orange cloud icon)
  6. Click Save

📝 Example: If you create a CNAME record with Name "claude-practice" pointing to "eduard15x.github.io", then claude-practice.devuard.com will show your GitHub Pages site.

Step 4: Configure Custom Domain in GitHub

Now tell GitHub about your custom domain:

Steps:

  1. Go to your GitHub repository
  2. Click Settings
  3. Go to Pages section
  4. Find "Custom domain" field
  5. Enter your subdomain (e.g., claude-practice.devuard.com)
  6. Click Save
  7. GitHub will verify the DNS record (this may take a few minutes)
  8. Check the "Enforce HTTPS" option (for security)

✅ Success: Your site is now accessible at your custom domain!

Troubleshooting Common Issues

❌ DNS not resolving / Site not accessible

  • Wait longer: DNS propagation can take up to 48 hours
  • Check CNAME record: Make sure the CNAME record is created correctly in Cloudflare
  • Verify GitHub Pages: Make sure your site works at the original GitHub URL first
  • Clear browser cache: Press Ctrl+Shift+Delete and clear cache, then try again

❌ GitHub shows "DNS check failed"

  • Wait a bit more: DNS needs time to propagate
  • Check spelling: Make sure your subdomain spelling matches exactly in both Cloudflare and GitHub
  • Verify CNAME is proxied: In Cloudflare, make sure the cloud icon is orange (Proxied), not gray

❌ HTTPS certificate not working

  • Check enforcement: Make sure "Enforce HTTPS" is checked in GitHub Pages settings
  • Wait for certificate: This can take 30 minutes to an hour
  • Cloudflare SSL: Make sure Cloudflare SSL/TLS is set to "Flexible" or "Full"

💡 Pro Tips

  • Use subdomains for multiple sites: You can create as many CNAME records as you want, all pointing to your GitHub Pages URL
  • Test locally first: Always test your GitHub Pages site before adding custom domain
  • Enable HTTPS: Always check "Enforce HTTPS" in GitHub settings for security
  • Keep it organized: Use meaningful subdomain names (claude-practice, aws-practice, etc.)
  • Save your DNS records: Keep a note of all your CNAME records for reference

✨ Summary

The complete workflow:

  1. Create GitHub repository with your code
  2. Enable GitHub Pages (you get a .github.io URL)
  3. Create CNAME record in Cloudflare (subdomain → github.io URL)
  4. Add custom domain in GitHub settings
  5. Wait for DNS to propagate
  6. Visit your custom domain - it works! 🎉
